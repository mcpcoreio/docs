## How connecting works

Every MCPCore server has a permanent HTTPS endpoint URL:

```
https://mcp.mcpcore.io/s/<your-server-id>
```

You copy this URL from the server's **Overview** tab and paste it into your AI client's MCP configuration. The client connects, calls `tools/list` to discover your tools and their schemas, and is ready to use them in conversations.

## Auth headers

If your server uses an authenticated security mode, include the appropriate header in the client configuration:

| Security mode | Header |
|---|---|
| Public | None needed |
| API Key | `X-API-Key: sk-xxxxxxxxxx` |
| OAuth 2.0 | Handled automatically by the client's OAuth flow |
| Bearer Token | `Authorization: Bearer <your_jwt>` |

---

## Other clients

MCPCore implements the standard MCP protocol over HTTP/SSE. Any MCP-compatible client can connect to your server using the endpoint URL.

The generic configuration pattern is:

```json
{
  "mcpServers": {
    "my-server": {
      "url": "https://mcp.mcpcore.io/s/<your-server-id>",
      "headers": {
        "X-API-Key": "sk-xxxxxxxxxx"
      }
    }
  }
}
```

Consult your client's documentation for the exact config file path and field names.
